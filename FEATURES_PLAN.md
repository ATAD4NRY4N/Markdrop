# Development Plan: CourseForge

**Goals**
Build a modern, full-featured eLearning authoring studio with a visual block editor, interactive assessments, SCORM 1.2/2004 export, adaptive learning paths, collaboration, and custom theming.

---

## Roadmap

### Phase 1: Core Editor Engine & Course Management
1. **Fix "Add Module" Bug [COMPLETED]:** Resolved initialization issues blocking course creation.
2. **Course Dashboard [COMPLETED]:** Built a comprehensive dashboard to view all courses, filter by draft/published, search by title/description, duplicate courses, toggle publish status, show module counts, and delete courses. Added `status` column via migration 002.
3. **Advanced Drag & Drop [COMPLETED]:** Seamless drag-and-drop of modules within the sidebar structure — intra-section reorder, cross-section drag, drag-to-unsectioned, `DragOverlay` ghost, droppable empty-section zones, touch sensor support. `persistSections` exposed from `CourseContext`.
4. **Copy & Paste Architecture [COMPLETED]:** Cross-slide and cross-course block copying and pasting. In-app clipboard singleton (`clipboard.js`) with synchronous helpers (`clipboardHasContent`, `getClipboardCount`). System clipboard integration with same-session in-app fallback. Clipboard count badge in the editor toolbar; per-block "paste after" button (shown when clipboard is loaded); `handleCopyModule` to copy all blocks of the active module; `handlePasteAfterBlock` to insert pasted blocks at a precise position. Copy/Paste module block actions in the sidebar panel dropdown. `pasteBlocksIntoModule` in `CourseContext` persists cross-slide pastes and syncs the active editor view via `blocks_json`-keyed `useEffect`.

### Phase 2: Layouts, Templates & Theming
1. **CSS GRID Support [COMPLETED]:** Fixed the orphaned `GridBlock` component (`onUpdate` signature bug repaired — now calls `onUpdate(block.id, updatedBlock)`). Added column weight presets (1:1, 1:1:1, 1:1:1:1, 2:1, 1:2, 1:2:1) via `block.weights` array and CSS `fr` units. Added a "layout" section to the AppSidebar palette with a "Columns / Grid" entry (double-click to add). Preview.jsx now renders `GridBlock` through the `ELEARNING_TYPES`/`ELEARNING_COMPONENTS` registry. Responsive: stacks to a single column on mobile (`max-sm:grid-cols-1`).
2. **Preconfigured Slide Templates [COMPLETED]:** Four preconfigured templates added to a new "Slide Templates" section at the bottom of AppSidebar: **Title Slide** (heading + learning objectives), **Two-Column** (H2 + 2:1 grid with text and image columns), **Quote Focus** (blockquote + knowledge check), **3-Step Timeline** (H2 + 1:1:1 grid). Clicking a template inserts all its blocks in sequence via `onBlockAdd`. Visible only in expanded sidebar state.
3. **Custom Themes System [COMPLETED]:** Per-course theme (heading font, body font, primary colour, accent colour, background colour) stored as `theme_json JSONB` in the `courses` table (migration `003_add_course_theme.sql`). Course Settings dialog expanded to two tabs — General (title/description) and Appearance (font selectors + colour pickers with live swatch preview). Theme applied in the inline Preview pane via injected CSS custom properties and a Google Fonts `@import`. Also threaded through `CoursePreview` (learner preview) and `buildPreviewHtml`/`buildScoHtml` in `scormUtils.js` so the exported SCORM package respects the chosen fonts and colours.

### Phase 3: Essential Interactive Blocks
1. **Callout / Admonition Block [COMPLETED]:** Markdown-based callouts with distinct UI states (Info, Warning, Success, Error, Tip, Note).
2. **Media Blocks [COMPLETED]:** 
   - Carousel Image Block with auto-play and pagination.
   - PDF Viewer Block with integrated accessibility and download options.
3. **Course Navigation Block [COMPLETED]:** Top/bottom progress bars, pagination buttons, and conditional progression (e.g., "locked until video watched").

### Phase 4: Advanced Interactive Assessments
1. **Refined MCQ / Knowledge Check [COMPLETED]:** Added `feedbackCorrect` and `feedbackIncorrect` text fields to KnowledgeCheckBlock, displayed after submission with color-coded correct/incorrect styling alongside the existing option highlight feedback.
2. **Fill in the Blank [COMPLETED]:** Standalone `FillInTheBlankBlock` with `___` placeholder syntax, ordered correct answers array, case-sensitive toggle, per-blank correction hints on submit, inline learner preview in editor mode, and custom correct/incorrect feedback messages.
3. **Matching Assessment [COMPLETED]:** `MatchingBlock` with term/definition pairs; learner clicks a term to select it then clicks the matching definition to pair — shuffled definitions, submit checks correctness, shows score and per-pair correct/incorrect highlights, with Try Again. Inline preview inside the editor.
4. **Hotspot Image Interaction [COMPLETED]:** `HotspotBlock` with click-on-image hotspot placement in editor, draggable x/y position inputs, label and popover content per spot. Learner view renders animated pin markers on the image; click reveals a popover with label and description.
5. **Categorization Drag & Drop [COMPLETED]:** Previously built `CategorizationBlock` with category buckets and item assignment.

### Phase 5: Localization workflows
1. **XLIFF Export/Import [COMPLETED]:** `src/lib/xliffUtils.js` extracts all translatable text fields from every block type (headings, paragraphs, alerts, lists, tables, flashcards, quizzes, fill-in-the-blank, matching, hotspot, categorization, branching, knowledge-check, learning-objectives, course-nav, progress-marker, and nested grid blocks) into XLIFF 1.2 XML with one `<file>` per module. Each `<trans-unit>` carries a stable ID (`{blockId}__{dot.path}`), the source text, an empty `<target/>` for the translator, and a human-readable `<note>`. `parseXliff()` reads the completed file back, and `applyTranslationsToModules()` deep-sets translated values via dot-path notation. `XliffDialog.jsx` (Export tab + Import tab) is mounted in `CourseBuilder.jsx` behind a **Translate** toolbar button: the Export tab downloads the XLIFF; the Import tab parses an uploaded file, shows a per-module translation summary, then creates a new course clone titled `{original} [{targetLang}]` with all translations applied.

### Phase 6: Collaboration & Review [COMPLETED]
1. **Collaboration Roles [COMPLETED]:** Three-tier role model: **Course Owner** (full edit, publish, and team management), **Editor** (can edit all modules), **Reviewer** (read-only learner preview + comment access). Stored in `course_collaborators` table (migration `004`). RLS policies extended on `courses` and `course_modules` so collaborators can SELECT (and editors can UPDATE) records they're invited to.
2. **Invitation System [COMPLETED]:** `course_invitations` table with UUID tokens, 7-day expiry, per-course-email uniqueness. `CollaboratorsDialog.jsx` mounted behind a **Team** toolbar button in `CourseBuilder`. Owner types an email, selects Editor or Reviewer, clicks "Invite" — a signed invite link is created and auto-copied to clipboard. A **Pending** tab shows outstanding invites with copy/revoke controls. `AcceptInvitePage.jsx` (`/accept-invite/:token`) verifies the token, shows the course name and role, and on "Accept" creates the `course_collaborators` record and redirects to the Reviewer view.
3. **Centralized Review Panel [COMPLETED]:** `ReviewPanel.jsx` (Sheet slide-over) accessible via a **Review** toolbar button in `CourseBuilder`. Supports: per-slide and course-wide comments; threaded replies; statuses (open / in-progress / resolved); priorities (low / normal / high); live filter by status and slide; text/author search. Comment cards show author avatar, timestamp, context badge (which slide/block), inline status change dropdown, and reply form. Authors and owners can delete; only owners can change status of others' comments.
4. **Reviewer View [COMPLETED]:** Standalone page at `/review/:courseId` — full-screen learner-faithful iframe preview (same `buildPreviewHtml` as Learner Preview) with slide navigation, module tab strip, and the ReviewPanel sheet. Role badge displayed for non-owners. Auth-gated: unauthenticated users are redirected to `/login?next=...`.
5. **Review Storage Layer [COMPLETED]:** `src/lib/reviewStorage.js` provides: `getCourseCollaborators`, `updateCollaboratorRole`, `removeCollaborator`, `getUserCourseRole` (returns owner/editor/reviewer/null), `inviteCollaborator` (upsert), `getPendingInvitations`, `cancelInvitation`, `getInvitationByToken`, `acceptInvitation` (token → collaborator record + accepted_at timestamp), `getCourseComments`, `createComment`, `updateComment`, `deleteComment`.

### Phase 7: Adaptive Learning [COMPLETED]
1. **Variants [COMPLETED]:** Define named learning branches (e.g. "Remediation Path", "Advanced Track") per course. Each variant is marked **required** (counts toward completion) or **optional** (enrichment only). Variant modules are invisible to learners until the variant is unlocked.
2. **Checkpoint Rules [COMPLETED]:** Any quiz or knowledge-check block can be designated a **checkpoint**. Score-based rules (`score ≥ N%` → unlock variant; `score < N%` → unlock variant) are evaluated in order — first match wins. A **fallback** fires if no rule matches. Rules are configured through the **Adaptive** toolbar panel (`AdaptivePanel.jsx`, `Sheet` slide-over).
3. **Adaptive Runtime in SCORM [COMPLETED]:** Each exported SCO page embeds `window._ADAPTIVE_CONFIG`, `window._ALL_MODULES_META`, and `window._CURRENT_FILE_INDEX`. The `ADAPTIVE_RUNTIME_JS` constant defines: `adaptiveInit()` (reads `cmi.suspend_data` JSON on load), `adaptiveSaveState()` (writes back after each evaluation), `adaptiveEvaluate(blockId, scorePercent)` (evaluates rules → unlocks variants → shows a toast notification), `adaptiveNavigate("next"|"prev")` (skips locked variant modules), and overrides for `nextModule()` / `prevModule()`. All state is serialized as `{"unlockedVariantIds":[…],"checkpointScores":{…}}` in `cmi.suspend_data`.
4. **Adaptive Utilities [COMPLETED]:** `src/lib/adaptiveUtils.js` — pure, framework-free helpers: `parseAdaptiveConfig`, `getCoreModules`, `getVisibleModules`, `buildModulesMeta`, `evaluateCheckpointRules`, `getRequiredVariantIds`, `computeCompletionInfo`, `getCheckpointableBlocks`.
5. **Authoring UI [COMPLETED]:** `AdaptivePanel.jsx` — Sheet panel with **Variants** tab (add/rename/delete variant, required toggle, module assignment checkboxes) and **Checkpoints** tab (per-block checkpoint cards; add/edit/delete score-based rules; fallback unlock; unsaved-change tracking with Save/Discard footer).
6. **Database [COMPLETED]:** Migration `005_add_adaptive_config.sql` adds `adaptive_config text NOT NULL DEFAULT '{"variants":[],"checkpoints":[]}'` to `public.courses`. Loaded and saved via `adaptiveConfig` state + `saveAdaptiveConfig` callback in `CourseContext`.

### Phase 8: Export & Analytics
1. **SCORM 1.2 / 2004 Polish:** Ensure packaged ZIP exports meticulously track `cmi.score.raw`, `cmi.core.lesson_status`, and `session_time`.
2. **LMS-Lite Analytics Dashboard:** For natively hosted courses, track sessions, active learners, completion rates, and average scores.

### Phase 9: Search & Replace [COMPLETED]
1. **Course-wide Search [COMPLETED]:** `src/lib/searchUtils.js` — pure search/replace engine that extracts all searchable text segments from every block type (headings, paragraphs, blockquotes, alerts, code, math, diagrams, lists, tables, image alt text, video/PDF titles, progress markers, course-nav labels, flashcard fronts/backs, fill-in-the-blank sentences and answers, matching pairs, knowledge-check prompts and options, hotspot labels and content, categorization items, learning objectives, branching choices, carousel captions, quiz questions/options/feedback). Handles grid nested blocks recursively.
2. **Search & Replace Dialog [COMPLETED]:** `src/components/blocks/CourseBuilderPage/SearchReplaceDialog.jsx` — floating dialog with a search input and a case-sensitive toggle (`Aa` button), a replace input, results grouped by module with match count badges, highlighted match context snippets using `<mark>`, keyboard navigation (Enter / Shift+Enter for next/prev match), **Replace** (single occurrence, replaces the selected match in place) and **Replace All** (bulk replace across every module in the course).
3. **Direct Navigation [COMPLETED]:** Clicking any search result switches to its module (via `setActiveModuleId`) and scrolls the matched block into the editor viewport with a 2-second violet outline pulse animation (`search-highlight-pulse` in `index.css`). A `data-block-id` attribute on each block wrapper in `Editor.jsx` allows the scroll target lookup.
4. **Keyboard Shortcut [COMPLETED]:** `Ctrl+H` / `Cmd+H` opens the Find & Replace dialog from anywhere in the builder, bypassing the input-field guard applied to other shortcuts. A Search toolbar icon button also opens the dialog.
5. **Replace Scope [COMPLETED]:** Replacing in the active module calls the existing `applyBlocks` pipeline (preserves undo history). Replacing in a background module persists directly via `updateModule` and updates context's `modules` state, keeping all modules in sync without switching the editor view.

### Phase 10: Accessibility [COMPLETED]
1. **WCAG Compliance Foundation [COMPLETED]:** CourseForge blocks are structured with semantic HTML, proper ARIA attributes, and heading hierarchy support to align with WCAG 2.1 inclusive-design principles. Interactive elements use correct roles and focus management. Accessibility is treated as an ongoing commitment across all platform updates.
2. **Alternate Text for Images [COMPLETED]:** The `ImageBlock`, `CarouselBlock`, and `HotspotBlock` all expose an `alt` field directly in the author editing UI. Alt text is included verbatim in SCORM export HTML (`<img alt="...">`) and the inline learner preview. Decorative images may use an empty alt to signal screen readers to skip. Alt text fields are included in XLIFF export for multilingual courses.
3. **Keyboard-Accessible Learning [COMPLETED]:** All interactive elements in the SCORM player and learner preview — buttons, quiz options, text inputs, navigation controls, and media players — are fully operable via keyboard. Focus rings are preserved. Tab order follows the logical visual layout. Assessment types (MCQ, fill-in-blank, matching, categorization, flashcards) support keyboard interaction without requiring a mouse click. The CourseBuilder itself exposes keyboard shortcuts (Ctrl+S, Ctrl+Z/Y, Ctrl+H, Ctrl+V) for efficient keyboard-driven authoring.
4. **RTL Languages Support [COMPLETED]:** The SCORM player and inline learner preview apply full right-to-left layout mirroring when a course is set to an RTL language. All content block types (headings, paragraphs, quizzes, interactive blocks) flow RTL. Navigation, sidebars, and progress indicators flip to match RTL reading order. SCORM export includes correct `lang` and `dir` HTML attributes so LMSs render courses natively. No duplicate course required — build once and localize to RTL.

Separate font assignments for headings, body, captions, and UI labels. Font settings saved as part of a reusable theme. Custom fonts bundled into SCORM export for offline LMS rendering.

Supports multiple brands, clients, or projects.
