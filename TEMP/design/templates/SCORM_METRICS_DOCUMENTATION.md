# SCORM Metrics Code System Documentation

## Overview

The SCORM Metrics Code System generates a unique, verifiable completion code for learners who complete all levels of the Space Game. This code contains all performance metrics and can be verified by instructors without requiring a centralized database.

## How It Works

### The Strategy: Encode, Sign, and Assemble 📦

1. **Data Collection**: As the learner progresses through levels, the game collects detailed metrics:
   - Level number and name
   - Time taken
   - Total attempts
   - Correct answers
   - Sectors explored
   - Space junk created
   - Success rate
   - Timestamp

2. **Encoding**: When all levels are complete, the metrics are:
   - Structured into a JSON array
   - Converted to Base64 encoding (compact text format)

3. **Signing**: A cryptographic signature is generated by hashing:
   - The learner's ID (from LMS)
   - The encoded payload
   - A secret "pepper" string (known only to course developers)

4. **Assembly**: The final code format is:
   ```
   [Base64EncodedMetrics].[8CharacterSignature]
   ```

## For Learners

### Getting Your Code

1. **Complete All Levels**: Play through all levels of the Space Game
2. **Victory Screen**: Upon completing the final level, you'll see:
   - Your final level statistics
   - A **🔐 Your Completion Code** section with a long code
3. **Copy the Code**: Click the "📋 Copy Code" button
4. **Submit**: Paste the code into your LMS assignment or send it to your instructor

### What the Code Contains

Your completion code is like a digital certificate that contains:
- Your learner ID (from the LMS)
- Performance data for every level you completed
- Timestamps proving when you completed each level
- A cryptographic signature that prevents tampering

**Important**: The code is unique to you and cannot be shared or reused by other learners!

## Technical Details

### Architecture

#### Client-Side (Game.md)

**SCORM API Detection:**
```javascript
// Automatically detects LMS environment
let scormAPI = null;
if (window.API) {
  scormAPI = window.API;  // LMS mode
} else {
  scormAPI = mockScormAPI;  // Local testing mode
}
```

**Metrics Collection:**
```javascript
// Saved after each level completion
const levelMetrics = {
  level: levelIndex + 1,
  levelName: levelName,
  timeTaken: stats.timeTaken,
  totalAttempts: stats.totalAttempts,
  correctAnswers: stats.correctAnswers,
  sectorsExplored: stats.sectorsExplored,
  spaceJunkCreated: stats.spaceJunkCreated,
  successRate: stats.successRate,
  timestamp: new Date().toISOString()
};
```

**Code Generation:**
```javascript
async function generateMetricsCode(gameData, learnerId) {
  // 1. Encode
  const jsonPayload = JSON.stringify(gameData);
  const base64Payload = btoa(jsonPayload);
  
  // 2. Sign
  const dataToSign = `${learnerId}:${base64Payload}:${COURSE_PEPPER}`;
  const hash = await sha256(dataToSign);
  const signature = hash.substring(0, 8);
  
  // 3. Assemble
  return `${base64Payload}.${signature}`;
}
```

#### Server-Side (verify_metrics_code.js)

**Verification Process:**
```javascript
function verifyAndDecodeMetricsCode(submittedCode, learnerId) {
  // 1. Split code
  const [submittedPayload, submittedSignature] = submittedCode.split('.');
  
  // 2. Recalculate signature
  const dataToSign = `${learnerId}:${submittedPayload}:${COURSE_PEPPER}`;
  const expectedHash = crypto.createHash('sha256').update(dataToSign).digest('hex');
  const expectedSignature = expectedHash.substring(0, 8);
  
  // 3. Verify
  if (submittedSignature !== expectedSignature) {
    return { isValid: false };
  }
  
  // 4. Decode
  const decodedJson = Buffer.from(submittedPayload, 'base64').toString('utf8');
  const gameData = JSON.parse(decodedJson);
  
  return { isValid: true, data: gameData };
}
```

### Cryptographic Components

**Pepper String:**
```javascript
const COURSE_PEPPER = 'scorm-ataccama-space-game-2025-q4';
```
- Static secret embedded in both client and verification tool
- Prevents code forgery without access to source
- Should be unique per course/semester

**Hash Function:**
- SHA-256 (Secure Hash Algorithm 256-bit)
- Browser: `crypto.subtle.digest()`
- Node.js: `crypto.createHash('sha256')`
- First 8 characters used as signature

**Encoding:**
- Base64 encoding for compact text representation
- Browser: `btoa()` / `atob()`
- Node.js: `Buffer.from(..., 'base64')`

## Security Considerations

### What This System Prevents

✅ **Random Code Submission**: Signatures cannot be guessed
✅ **Code Sharing**: Each code is tied to a specific learner ID
✅ **Metric Tampering**: Any change invalidates the signature
✅ **Replay Attacks**: Timestamps prove completion time

### What This System Does NOT Prevent

⚠️ **Course Pepper Exposure**: If a learner finds the pepper in source code, they could forge codes
⚠️ **Browser DevTools Manipulation**: Skilled users could modify game state before code generation
⚠️ **Collaborative Cheating**: One learner playing while logged in as another

### Mitigation Strategies

1. **Obfuscate Code**: Use JavaScript minification/obfuscation in production
2. **Rotate Peppers**: Change the pepper each semester
3. **Cross-Reference**: Compare completion times with LMS access logs
4. **Manual Review**: For suspicious submissions, ask learners to demonstrate knowledge

## Troubleshooting

### Common Issues

#### "Code verification failed - Signature mismatch"

**Causes:**
- Learner ID mismatch between LMS and verification
- Code was copied incorrectly (missing characters)
- Pepper string differs between Game.md and verification script

**Solutions:**
1. Verify the learner ID matches LMS records exactly
2. Ask learner to copy code again using the Copy button
3. Check that `COURSE_PEPPER` is identical in both files

#### "Invalid code format"

**Causes:**
- Code truncated during copy/paste
- Code split across multiple lines

**Solutions:**
1. Ensure the entire code is on a single line
2. Check for extra spaces or line breaks
3. Verify code contains exactly one dot (`.`) separator

#### "Verification error: Unexpected token"

**Causes:**
- Base64 payload is corrupted
- JSON structure was modified

**Solutions:**
1. Have learner regenerate code by completing game again
2. Manually decode Base64 to inspect JSON structure:
   ```bash
   node -e "console.log(Buffer.from('PAYLOAD_HERE', 'base64').toString())"
   ```
